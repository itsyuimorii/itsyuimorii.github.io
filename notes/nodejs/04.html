# ⛳️ 04. npm & package

# 1. package

# 1.1 Overview

Since Node is a light kernel platform, it provides a series of built-in modules, but they are not enough to meet the needs of developers, so the concept of packages emerged: similar to core modules, they are pre-designed functions or APIs wrapped into a folder and made available to developers.

Node itself doesn't have many functional APIs, so there are a lot of packages developed by third-party developers on the market.

# 1.2 Package loading mechanism

Node.js uses the CommonJs modularity mechanism for third-party packages downloaded through npm. We introduce third-party packages in our projects as: let xx = require('third-party package name'). What exactly is the principle mechanism of the require method for loading third-party packages, let's explore it today.

  1. require('third-party package name') gives priority to finding third-party packages in the sibling directory node_modules of the module that loads the package. 2.
  2. Find the package.json file in the third-party package, and find the entry module corresponding to the main attribute in it, which is the loaded third-party module.
  3. If the package.json file is not found in the third-party package to be loaded or there is no main attribute in the package.json file, the index.js file in the third-party package will be loaded by default.
  4. If the node_modules folder is not found at the same level as the file where the third-party module is loaded, or if none of the above is found, the node_modules folder will be searched for in the parent directory at the next level, with the same rules as above.
  5. If the module's disk root path is not found, an error will be reported: can not find module xxx.

# 1.4 The concept of npm

NPM: Node Package Manager. official link: https://www.npmjs.com/

Node.js has developed into a very large ecosystem by now. Once the package ecosystem flourishes, there must be tools to manage these packages. npm was born.

For example, when we develop in Java, we need to use the built-in libraries provided by theJDK, as well as third-party libraries. Similarly, when developing in JS, we can use the NPM package manager to easily use mature and excellent third-party frameworks and integrate them into our own projects, greatly accelerating the daily development build process.

Over time, two layers of NPM concepts have emerged.

  • One is Node's open module registry and management system
  • The other layer is Node's default module manager, a command-line software that installs and manages Node modules.

# 1.5 Installation of npm (no separate installation required)

NPM does not need to be installed separately. By default, when you install Node, you install NPM along with it

Type npm -v to see the version of npm

Also, the NPM that comes with Node may not be the latest version, so you can update to the latest version with the following command.

$ npm install npm -g

# 6. Use nodemon to restart the service automatically

In the process of development, we must restart the server after each code change, obviously this operation is relatively inefficient, here to recommend a tool, nodemon, nodemon can help us listen to the project in real-time code changes, and automatically restart the service, and the configuration is simple.

  1. install: npm install -g nodemon

    If you can't use nodemon, you have to configure it in environment variables.

  2. Run the project with nodemon, replacing the previous node app.js.

nodemon app.js
1

Once the project is running, nodemon will automatically listen for code changes and restart the service, greatly increasing our development efficiency.

  1. nodemon common configuration
  • Specify the port number of the application on the command line: nodemon . /server.js localhost 8080
  • Check the help, there are many options inside the help that are clear at a glance: nodemon -h or nodemon --help
  • Run debug mode: nodemon --debug . /server.js 80
  • Manually restart the project: Nodemon can be restarted manually by typing rs in the terminal window where the Nodemon command is run, and then hitting enter.

# 8. package.json file

# Overview

  1. too many folders and files are too fragmented, when we copy the whole project to others, the transfer speed will be very slow and slow.
  2. complex module dependencies need to be recorded to ensure that the module version and the current consistent, otherwise it will cause the current project to run error

# package.json file

Project description file, which records the current project information, such as project name, version, author, github address, which third-party modules the current project depends on, etc.

It is generated using the npm init -y command.

# Project Dependencies

  • The third-party packages that you need to rely on during the development and online operation phases of a project are called project dependencies
  • Files downloaded with the npm install package name command are added to the dependencies field of the package.json file by default

# Development Dependencies

  • Third-party packages that need to be relied on during the development phase of the project but not during the online operation phase are called development dependencies
  • Use the npm installpackage name -save-dev command to add the package to the devDependencies field of the package.json file
npm i nodemon --save-dev
1

# Role of the package-lock.json file

  • Lock the package version to ensure that no problems arise when downloading again due to different package versions
  • Speed up the download, because the tree structure of the third-party packages that the project depends on and the package download addresses are already recorded in this file, so you can just download them when reinstalling, no extra work is needed

# 9. Loading mechanism of modules in Node.js

# Module lookup rules: When the module has a path but no suffix

require('./find.js');
1
  1. require method to find the module according to the module path, if it is the full path, directly introduce the module.
  2. if the module suffix is omitted, first find the JS file with the same name and then find the JS folder with the same name
  3. if you find the folder with the same name, find the index.js in the folder
  4. if there is no index.js in the folder, it will look for the entry file in the main option in the package.json file in the current folder
  5. If the specified entry file does not exist or if no entry file is specified, an error will be reported and the module will not be found

# Module lookup rules: When the module has no path and no suffix

require('find');
1

  1. Node.js will assume it is a system module

    Node.js will go to the node_modules folder

    first to see if there is a JS file by that name

    then see if there is a folder with that name

    If it's a folder, see if there's an index.js in it

    If there is no index.js check the main option in package.json in that folder to determine the module entry file

    Otherwise, the error is not found

# 10. Confliction solution

npm WARN old lockfile The package-lock.json file was created with an old version of npm #

To resolve the warning "npm WARN old lockfile The package-lock.json file was created with an old version of npm":

  1. Run the npm install --package-lock-only command to resolve any conflicts and update your package-lock.json file.
  2. Delete your node_modules and package-lock.json.
  3. Install your modules with npm install.
  4. Commit the updated package-lock.json file to your repository.
